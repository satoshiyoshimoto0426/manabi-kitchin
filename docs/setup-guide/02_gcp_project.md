# 02. GCP プロジェクト・請求・サービスアカウント

> 所要時間: **40分**
> 費用: **¥0** (無料枠内の使用想定、初回$300クレジット付与あり)
> このガイドで取得するもの: `GCP_PROJECT_ID`, サービスアカウントJSONファイル

---

## 📝 このガイドでやること

1. GCP (Google Cloud Platform) にアクセス
2. 請求先アカウント (クレジットカード登録) を作成
3. 「manabi-ops」プロジェクトを作成
4. 必要な API を一括で有効化
5. サービスアカウントを作成して JSON キーをダウンロード
6. **月額予算アラート**を設定 (超重要)

### ゴール

✅ GCP のプロジェクト `manabi-ops` が作られている
✅ プログラムが GCP を操作するための「ロボット用ID」(サービスアカウント) が用意されている
✅ 予期せぬ高額請求を防ぐアラートが設定されている

---

## 🚶 手順

### Step 1. GCP にアクセス

1. ブラウザで以下を開く
   👉 **https://console.cloud.google.com/**
2. Gmail アカウントでログイン (ガイド00で準備したもの)
3. 初回は「利用規約に同意」のチェックボックスに ✅ → 「同意して続行」

---

### Step 2. 請求先アカウントを作成 (初回のみ)

> 💳 **重要**: クレジットカードが必要ですが、**初回は $300 (約4.5万円) の無料クレジットが90日間自動付与**されます。この範囲なら一切課金されません。

1. 画面左上のメニュー (☰) → **「お支払い」** をクリック
2. **「請求先アカウントを追加」** または **「請求先アカウントを作成」** をクリック
3. 情報を入力:

   | 項目 | 入力する値 |
   |------|-----------|
   | **アカウント種別** | 個人 or ビジネス (団体の場合はビジネス) |
   | **名前** | 代表者氏名 or 団体名 |
   | **住所** | 団体住所 |
   | **電話番号** | 連絡可能な番号 |
   | **支払い方法** | クレジットカード情報 |

4. **「送信して課金を有効にする」**

> 📌 $300クレジットが付与されると「Free Trial credit: $300.00」という表示が出ます。

---

### Step 3. プロジェクトを作成

**プロジェクトとは**: GCP でサービスをまとめる「部屋」のようなもの。

1. 画面上部の青いバーの左側にある **「プロジェクトを選択」** のプルダウンをクリック
2. 表示されるダイアログの右上の **「新しいプロジェクト」** をクリック
3. 以下を入力:

   | 項目 | 入力する値 |
   |------|-----------|
   | **プロジェクト名** | `manabi-ops` |
   | **プロジェクト ID** | 自動生成 (例: `manabi-ops-432501`) → **これをメモ帳に記録** |
   | **組織** | 選択不要 (なしでOK) |
   | **場所** | 選択不要 |

4. **「作成」** をクリック → 30秒ほど待つ
5. 通知アイコンに「プロジェクトが作成されました」と表示されたら、画面上部のプロジェクト名を **「manabi-ops」** に切り替える

メモ帳に追記:
```
=== GCP ===
GCP_PROJECT_ID = manabi-ops-432501  (ここは皆さんのIDに置き換え)
```

---

### Step 4. 請求先アカウントをプロジェクトに紐づけ

1. 画面左メニュー (☰) → **「お支払い」** をクリック
2. **「請求先アカウントをリンク」** をクリック
3. Step 2 で作った請求先アカウントを選択して **「アカウントを設定」**

> ⚠ これをやらないと Cloud Run デプロイ時にエラーになります。

---

### Step 5. 月額予算アラートを設定 (超重要 💰)

> 🚨 **必ずやってください**。これをやらないと、万一バグで無限ループが発生した時に高額請求が来ます。

1. 左メニュー → **「お支払い」** → **「予算とアラート」**
2. **「予算を作成」** をクリック
3. 入力:

   | 項目 | 入力する値 |
   |------|-----------|
   | **名前** | `manabi-ops月額予算` |
   | **予算額** | **10000 円** (月額) |
   | **対象期間** | 月次 |
   | **プロジェクト** | manabi-ops |

4. **「次へ」**
5. 通知しきい値を以下に設定:
   - **50%** で通知 (¥5,000到達時)
   - **90%** で通知 (¥9,000到達時)
   - **100%** で通知 (¥10,000到達時)
6. 通知メールに **Gmail アドレス** を入力
7. **「完了」**

これで予算の 50%, 90%, 100% に達した時点で Gmail にアラートが届きます。

---

### Step 6. 必要な API を有効化

ManabiOps が使う Google のサービスを一括で「使えるようにする」作業です。

1. 左メニュー → **「API とサービス」** → **「ライブラリ」**
2. 以下を**1つずつ**検索して、それぞれ **「有効にする」** をクリック:

| # | API 名 | 検索ワード | 用途 |
|---|--------|-----------|------|
| 1 | **Cloud Run Admin API** | `cloud run admin` | アプリの実行 (ガイド09) |
| 2 | **Cloud Build API** | `cloud build` | デプロイ時のビルド |
| 3 | **Artifact Registry API** | `artifact registry` | Dockerイメージ保管 |
| 4 | **Firestore API** | `firestore api` | データベース (ガイド05) |
| 5 | **Document AI API** | `document ai` | OCR (ガイド04) |
| 6 | **Generative Language API** | `generative language` | Gemini AI (ガイド03) |
| 7 | **Cloud Storage API** | `cloud storage api` | ファイル保管 (ガイド06) |
| 8 | **Google Sheets API** | `google sheets api` | 会計台帳 (ガイド07) |
| 9 | **Secret Manager API** | `secret manager` | パスワード保管 (ガイド08) |
| 10 | **Cloud Scheduler API** | `cloud scheduler` | バッチ実行 (ガイド11) |
| 11 | **Cloud Logging API** | `cloud logging` | ログ確認 |

> 📌 各 API は有効化に 30秒〜1分 かかります。有効化完了を待ってから次のAPIに進んでください。

### 一括コマンド版 (上級者向け)
Cloud Shell が使える方は、画面右上の ">_" マーク (Cloud Shell) を開いて以下を貼り付けると一発で有効化できます:

```bash
gcloud services enable \
  run.googleapis.com \
  cloudbuild.googleapis.com \
  artifactregistry.googleapis.com \
  firestore.googleapis.com \
  documentai.googleapis.com \
  generativelanguage.googleapis.com \
  storage.googleapis.com \
  sheets.googleapis.com \
  secretmanager.googleapis.com \
  cloudscheduler.googleapis.com \
  logging.googleapis.com
```

---

### Step 7. サービスアカウントを作成

**サービスアカウントとは**: プログラム (Cloud Run) が GCP を操作するための「ロボット用ID」。

1. 左メニュー → **「IAM と管理」** → **「サービスアカウント」**
2. 上部の **「＋ サービスアカウントを作成」** をクリック
3. 入力:

   | 項目 | 入力する値 |
   |------|-----------|
   | **サービスアカウント名** | `manabi-ops-runner` |
   | **サービスアカウントID** | 自動 (manabi-ops-runner) |
   | **説明** | `ManabiOps の Cloud Run 実行用` |

4. **「作成して続行」**
5. **「ロールを選択」** で以下を**全て追加** (1つずつ選択 → 「別のロールを追加」を繰り返し):

   | ロール名 | 用途 |
   |---------|------|
   | **Cloud Datastore ユーザー** | Firestore 読み書き |
   | **Storage オブジェクト管理者** | Cloud Storage 読み書き |
   | **Document AI API ユーザー** | DocumentAI 呼び出し |
   | **Secret Manager のシークレット アクセサー** | Secret 取得 |
   | **ログ書き込み** | Cloud Logging |

6. **「続行」** → **「完了」**

---

### Step 8. サービスアカウントの JSON キーをダウンロード

> ⚠ このJSONファイルは**超重要**。厳重管理してください。

1. 作成されたサービスアカウント `manabi-ops-runner@...` をクリック
2. 上部タブ **「キー」** をクリック
3. **「鍵を追加」** → **「新しい鍵を作成」**
4. **キーのタイプ: JSON** を選択 → **「作成」**
5. JSON ファイルが自動ダウンロードされる (例: `manabi-ops-432501-abc123.json`)

### JSONファイルの扱い

- [ ] ファイル名を `manabi-ops-sa-key.json` にリネーム
- [ ] 運営団体の Google Drive の**共有されていない**フォルダに保存
- [ ] パスワードマネージャーにバックアップを保存 (1Password の「セキュアノート」に添付など)
- [ ] **絶対に Slack / LINE / Gmail 等に貼り付けて送らない**
- [ ] **絶対に GitHub にコミットしない**

メモ帳に追記:
```
=== GCP サービスアカウント ===
メールアドレス: manabi-ops-runner@manabi-ops-432501.iam.gserviceaccount.com
JSON キー保存場所: (Google Drive の URL やパス)
```

---

### Step 9. 動作確認 (Cloud Shell で)

画面右上のヘッダーに **「>_」(Cloud Shell)** アイコンがある → クリックして起動

```bash
# プロジェクトを確認
gcloud config list project

# 有効な API 一覧
gcloud services list --enabled | grep -E "(run|firestore|documentai|sheets|storage|secretmanager)"

# サービスアカウント一覧
gcloud iam service-accounts list
```

manabi-ops-runner が表示されれば OK。

## 🆘 よくあるトラブル

### Q1. クレジットカード登録でエラーが出る
- デビットカードやプリペイドカードは使えません (クレジットカードのみ)
- カード会社の不正利用検知に引っかかっている可能性 → カード会社に連絡

### Q2. プロジェクト作成で「組織が必要」と出る
- 個人アカウントなら「組織」を選択せず、「組織なし」のままでOK

### Q3. 「$300 無料クレジット」がどこにあるかわからない
- 左メニュー → お支払い → 概要 の右側に **「無料トライアル」** というセクションがあります

### Q4. API有効化で「権限がありません」と出る
- プロジェクトを切り替えているか確認 (画面上部の青いバーの左に「manabi-ops」と出ているか)

### Q5. サービスアカウント JSON をうっかり GitHub に公開してしまった
1. 即座に GCP Console でそのキーを**削除** (IAM → サービスアカウント → 鍵)
2. 新しい鍵を作成
3. 既に使われていないか Cloud Logging を確認
4. 詳しくは開発担当者に相談
