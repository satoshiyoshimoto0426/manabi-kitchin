# 06. Cloud Storage バケット作成

> 所要時間: **15分**
> 費用: **5GB まで無料** — まなびキッチン規模 (月10GB想定) で **月¥15〜¥30**
> このガイドで取得するもの: `GCS_BUCKET_NAME` = **`manabi-ops-media`**

---

## 📝 このガイドでやること

Cloud Storage は ManabiOps が **LINE で受信した画像・動画・OCR前のレシート原本** を保存するストレージです。
ここでは専用バケット (=フォルダのようなもの) を1つ作成します。

### ゴール

✅ `manabi-ops-media` という名前のバケットが作成される
✅ リージョンは `asia-northeast1 (東京)`
✅ ライフサイクルルール (1年後自動削除) が設定される → 要件定義書 7.2 準拠

---

## 🎯 重要な用語

| 用語 | 意味 |
|------|------|
| **バケット** | ストレージの「大きなフォルダ」。1プロジェクトに複数作れる |
| **オブジェクト** | バケット内のファイル本体 (画像1枚 = 1オブジェクト) |
| **Standard Storage** | 標準ストレージ (今回はこれ) |
| **ライフサイクル** | 「○日後に自動削除」等のルール |

---

## Step 1. Cloud Storage コンソールを開く

1. [GCP Console](https://console.cloud.google.com/) でプロジェクトが **`manabi-ops`** になっていることを確認
2. 画面左メニュー (☰) → **「Cloud Storage」** → **「バケット」**
   - または上部検索バーで `Cloud Storage` と検索

---

## Step 2. バケットを作成

1. 画面上部の **「+ 作成」** (CREATE) ボタンをクリック

### 2-1. バケット名

- **名前**: `manabi-ops-media`
- ⚠ **全世界でユニーク**である必要があります (既に他者が使っている場合は `manabi-ops-media-001` のように数字を追加)
- 使える文字: 小文字英数字・ハイフン・アンダースコア・ピリオド
- **「続行」** をクリック

### 2-2. データの保存場所

- ロケーションタイプ: **「Region」** (⭐ Region を選択 — Multi-region は高コスト)
- リージョン: **`asia-northeast1 (Tokyo)`** ⭐ 東京
- **「続行」** をクリック

### 2-3. ストレージクラス

- **「Standard」** (標準) を選択 (デフォルト)
- ❌ Nearline / Coldline / Archive は選ばない (アクセス頻度が低いデータ向け)
- **「続行」** をクリック

### 2-4. アクセス制御

- **「均一」** (Uniform) を選択 ⭐
  - ❌ 「きめ細かい管理」(Fine-grained) は選ばない
- **「公開アクセス禁止を適用する」** にチェック ✅ (⭐ 重要 — 画像は公開しない)
- **「続行」** をクリック

### 2-5. データの保護

- オブジェクトのバージョニング: **オフ** (OFF) のまま (コスト節約)
- 保持ポリシー: **設定しない**
- **「作成」** をクリック

### 2-6. 公開アクセス警告が出たら

- 「このバケットへの公開アクセスはブロックされます」と出たら **「確認」** をクリック

→ **バケットが作成されました** 🎉

---

## Step 3. ライフサイクルルール設定 (要件定義書 7.2 準拠)

要件定義書 7.2「メディアは1年で自動削除」を実現します。

1. 作成したバケット `manabi-ops-media` をクリック
2. 上部タブの **「ライフサイクル」** (Lifecycle) をクリック
3. **「+ ルールを追加」** (ADD A RULE) をクリック

### ルール1: 1年後に削除

- **アクション**: **「オブジェクトを削除」** (Delete object)
- **「続行」** をクリック
- **オブジェクトの条件を選択**:
  - ☑ **「年齢」** (Age) にチェック
  - 値: **`365`** (日)
- **「作成」** をクリック

→ これで **1年経過したファイルは自動削除** されます (人手不要)

---

## Step 4. フォルダ構造は自動作成される

ManabiOps は以下のフォルダ構造に自動でファイルを保存します。
**手動作成は不要**です。

```
manabi-ops-media/
├── receipts/          ← レシート画像 (OCR前の原本)
├── photos/            ← イベント写真 (オリジナル)
├── photos_blurred/    ← 顔ぼかし済み写真
├── videos/            ← 動画 (リール素材)
├── videos_edited/     ← 編集済みリール動画
└── rosters/           ← 名簿画像
```

## 🆘 トラブルシューティング

### ❌ 「このバケット名は既に使用されています」

- 名前は **全世界で1つ** のため、`manabi-ops-media-2026` のように年号や数字を追加
- 最終的な名前を `.env` の `GCS_BUCKET_NAME` に設定する

### ❌ 作成後、料金が心配

- 無料枠: **5GB/月 + 5000回の書込無料**
- ライフサイクル365日で自動削除するため無限に蓄積することはありません
- 月10GB程度のアップロードで **月¥15〜¥30** 程度

### ❌ 誤ってバケットを削除してしまった

- バケット内に1ファイルもなければすぐに同名で再作成可能
- ファイルがあった場合、削除から最大30日は復旧可能な場合あり → GCP サポートへ
