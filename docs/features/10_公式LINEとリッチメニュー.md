# 公式LINE × ManabiOps：統合の仕組みとリッチメニュー設計

> 対象読者：運営代表者、スタッフ、ボランティア
> 読了時間：約8分
> 関連ドキュメント：`02_全体像と使い方フロー.md`, `06_管理画面の使い方.md`

## 1. 結論：公式LINE統合は **すでに前提** です

ManabiOpsは「LINE公式アカウント（旧：LINE@）」を入口にして動く設計です。
ですので「統合できますか？」の答えは **はい、というよりLINE無しでは動きません**。

```
┌─────────────────┐         ┌──────────────┐         ┌─────────────┐
│ スタッフのLINE   │ ──────▶ │ LINE公式      │ ──────▶ │ ManabiOps   │
│ (個人アカウント) │ 友だち  │ アカウント    │ Webhook │ (Cloud Run) │
└─────────────────┘         └──────────────┘         └─────────────┘
                                                            │
                                                            ▼
                                                     AI処理 + 自動記録
```

### 必要なLINE公式アカウントの種類
- **Messaging API対応の公式アカウント**（無料プランでOK）
- フリープラン：月200通まで無料 → 学び舎キッチン規模なら十分
- 有料化のタイミング：月3,000通超えるあたり（≒ 月¥5,000〜）

### 何ができるようになるか
| 機能 | LINE公式単体 | LINE公式 × ManabiOps |
|------|------------|---------------------|
| 写真受信 | ✗ 自動処理不可 | ✅ AI自動分類 |
| 自動応答 | △ キーワードだけ | ✅ 文脈理解＆Flex Message |
| リッチメニュー | △ 手動でタップ → 定型文返信 | ✅ ワンタップで業務開始 |
| ユーザー管理 | △ ブロック/通報のみ | ✅ Owner/Staff/Viewer権限 |
| 帳票生成 | ✗ 不可 | ✅ Sheets/Firestore自動連携 |

## 2. リッチメニューとは？（おさらい）

LINEトーク画面の **下半分に常に出ている「ボタン群」** のことです。
スマホ画面の貴重な領域を専有するので、**1日に何回も使う動作だけ** を厳選します。

### 良いリッチメニューの3原則

1. **2タップ以内で完了する** — 深い階層は使わない
2. **絵で分かる** — 文字が小さくても直感的
3. **役割に合わせて切り替わる** — Owner / Staff で別メニュー

## 3. ManabiOps 推奨リッチメニュー設計（2層構成）

### 🟢 メインメニュー（全Staffに表示）：3列×2段＝6ボタン

```
┌─────────────┬─────────────┬─────────────┐
│  🧾 領収書  │  📋 名簿撮影 │  📷 活動写真 │
│   送信      │   送信       │   送信       │
├─────────────┼─────────────┼─────────────┤
│  📊 月次    │  📮 保留中の │  ⚙ 管理画面 │
│  サマリー    │   承認待ち   │  /  ヘルプ   │
└─────────────┴─────────────┴─────────────┘
```

| ボタン | タップ時の動作 | 利用頻度 |
|--------|--------------|---------|
| 🧾 領収書送信 | 「📷カメラを起動 → 撮影 → 送る」を促すテキスト＋カメラ起動ガイド | 週3-4回 |
| 📋 名簿撮影 | 同上、名簿用の構図ヒント表示 | 週1回（活動日） |
| 📷 活動写真 | 「複数枚OK / 動画OK」と説明＋送信誘導 | 週1回（活動日） |
| 📊 月次サマリー | サーバ側で集計→Flex Messageで返答 | 月2-3回 |
| 📮 承認待ち | 自分宛の承認リクエスト一覧表示 | 1日2-3回 |
| ⚙ 管理画面/ヘルプ | Owner→管理画面リンク / Staff→ヘルプテキスト | 月1-2回 |

**なぜ6つか？**
- 7つ以上は人間の短期記憶（マジックナンバー7±2）を超える → 探す時間が増える
- 6つだと縦持ちスマホで親指が全ボタンに届く（Fittsの法則）
- 1つあたり最低 **125px×125px** の指タップ領域（Apple HIG準拠）

---

### 🟠 Owner専用メニュー（代表者だけ切り替え可能）：3列×2段＝6ボタン

```
┌─────────────┬─────────────┬─────────────┐
│  👥 メンバー │  📊 詳細    │  💰 経費    │
│   管理      │   ダッシュ   │   レポート  │
├─────────────┼─────────────┼─────────────┤
│  📱 SNS投稿 │  ⚠ アラート │  🔄 通常    │
│   履歴      │   一覧      │   メニューへ │
└─────────────┴─────────────┴─────────────┘
```

Ownerは右下「🔄 通常メニューへ」でメインに戻れます。

## 4. 各ボタンの中身を詳細設計

### A. 🧾 領収書送信 ボタン
**Postback action**: `action=guide&type=receipt`

タップ → 以下のFlex Messageが返信される：

```
┌──────────────────────────────┐
│ 🧾 領収書を送るには         │
│                              │
│ 1. このメッセージの「📎」    │
│    をタップ                  │
│ 2. 「カメラ」を選ぶ          │
│ 3. 領収書全体が枠内に収まる  │
│    ように撮影                │
│ 4. 送信ボタンを押す          │
│                              │
│ 💡 ヒント                    │
│ • 複数枚OK（連続で送れます） │
│ • 暗いと読み取り精度↓        │
│ • 折れ目はできるだけ伸ばして │
│                              │
│ [📷 カメラを起動] ← URIボタン│
└──────────────────────────────┘
```

URIボタンは `line://nv/camera/` のディープリンクで直接カメラを起動できます（iOS/Android両対応）。

### B. 📋 名簿撮影 ボタン
同様の構造。「氏名欄が読める明るさで」「真上からの撮影を推奨」のヒント表示。

### C. 📷 活動写真 ボタン
「複数枚一気に送ってOK」「動画は最大15秒推奨」「全員が映っていなくてOK」のヒント。

### D. 📊 月次サマリー ボタン
**Postback action**: `action=summary`
タップ → サーバ側が即座に当月集計を返信（既存の `formatSummaryText` を利用）。

### E. 📮 承認待ち ボタン
**Postback action**: `action=pending`
タップ → 自分宛の保留中リクエスト一覧（既存の `pending` ハンドラを活用）。

### F. ⚙ 管理画面 / ヘルプ ボタン
- Owner判定 → 管理画面URL（LINE LIFF）に遷移
- Staff判定 → 詳細ヘルプFlex Messageを返信

## 5. 「賢い切り替え」の仕組み

LINE Messaging APIには `linkRichMenuIdToUser` という機能があり、**ユーザーごとに別のリッチメニュー** を表示できます。

```typescript
// ログイン時 (友だち追加時)
if (user.role === 'owner') {
  await client.linkRichMenuIdToUser(userId, OWNER_MENU_ID);
} else if (user.role === 'staff') {
  await client.linkRichMenuIdToUser(userId, STAFF_MENU_ID);
} else {
  // viewer / unauthorized
  await client.unlinkRichMenuFromUser(userId);
  // → デフォルトの「シンプルメニュー」が表示される
}
```

これで：
- 代表者には全機能を
- スタッフには日常作業だけを
- 部外者には何も表示されない（=セキュリティ）

## 6. デザインのポイント

### 配色（既存ブランドに合わせる）
- メイン背景：温かいクリーム色 `#FFFDF7`
- アクセント：オレンジ `#F57C00`（活動の温かさ）
- ボタン背景：ブランドグリーン `#2E7D32` の薄い色
- 文字：濃いグリーン `#1B5E20`

### サイズ仕様（LINE公式準拠）
- フルサイズ：2500×1686px（推奨）
- ハーフサイズ：2500×843px（メニュー2段なら使える）
- ファイル形式：JPG または PNG（1MB以下）

### アイコン選定基準
- 絵文字＋日本語の **ダブル表記** で誰でも分かる
- アイコンは **塗りつぶし＞線画**（小さい画面で視認性◎）
- 色は **2色まで**（情報過多を防ぐ）

## 7. 運用 Tips：こども食堂ならではの工夫

### 💡 Tip 1: 活動日の朝だけ表示するメニュー
活動日の8時に「今日のメニューに切り替え」を実行し、終了後にデフォルトに戻す。
→ Cloud Schedulerで自動化（週1度の実行）。

```
活動日メニュー：
┌─────────────┬─────────────┐
│  🍚 食材    │  📋 出席    │
│   買い出し   │   名簿      │
├─────────────┼─────────────┤
│  📷 活動    │  ✨ 終了    │
│   写真      │   報告      │
└─────────────┴─────────────┘
```

### 💡 Tip 2: 新人ボランティアには「練習用」を最初にリンク
- 練習用メニュー：「✋ご挨拶」「📖 マニュアル」「❓ 質問する」のみ
- 1週間 慣れたら 通常メニューに自動切り替え（Firestoreにフラグ）

### 💡 Tip 3: 「困った時」ボタンは右下固定
- 心理的安全性のため、いつでも逃げ道がある状態を作る
- タップ → スーパーバイザーに直接通知（Slack的に）

## 8. セットアップ手順（5分でできる）

詳細は `docs/setup-guide/15_richmenu_setup.md` を参照。
要約：

1. リッチメニュー画像 3枚を準備（main / owner / activity_day）

## 9. よくある質問

### Q1. リッチメニューは何回でも変更できますか？
**A**. はい、無制限です。デザインを変えても画像差し替え＋API1回呼ぶだけです。

### Q2. iOSとAndroidで見た目変わりますか？
**A**. ほぼ同じですが、AndroidはOSフォントの影響で日本語の太さが微妙に違います。**ボタン内の文字は画像に焼き込む** ことで完全統一できます（推奨）。

### Q3. 既存のLINE公式アカウントを使えますか？
**A**. はい、Messaging API対応に切り替えるだけです。友だちはそのまま引き継げます。

### Q4. 友だち以外には機能しますか？
**A**. 友だち追加が必須です。逆に **追加しない=部外者にはアクセス不可** という安心設計でもあります。

### Q5. リッチメニューで業務以外の連絡（雑談）も妨げない？
**A**. リッチメニューは下半分のみ。上半分の入力欄からは普通にテキストチャットできます。
