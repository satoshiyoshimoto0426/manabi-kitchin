## 🎓 助成金での計上方法

### 計上可能な項目

- **事業運営費**: 家計簿ソフト相当として
- **システム利用料**: 月額利用料として
- **広報費**: SNS 運用ツールとして

### 必要書類

- GCP 月次請求書 (自動メール送付)
- 領収書 (GCP Console からDL可)
- 運用担当者の業務記録

### 対応している助成金 (一般例)

- 赤い羽根共同募金 (中央共同募金会)
- 子どもの未来応援基金
- 地域自治体の子ども食堂支援事業
- 企業のCSR助成金 (対象外もあり)

**個別の可否は各助成団体に確認を推奨**。

## 📈 料金を安く抑えるコツ

### 1. Cloud Run を「無料枠運用」する

```bash
# min-instances = 0 にする (常時起動しない)
--min-instances 0
```

- リクエストがない時は自動停止 (無料)
- 最初のリクエスト時のみ「コールドスタート」(3〜5秒遅延)
- こども食堂の利用頻度なら、コールドスタートは許容範囲

### 2. Cloud Storage のライフサイクル設定

```
1年後に自動削除
  → 永遠に溜まらない
  → 5GB無料枠内で運用可
```

### 3. Document AI の精度を上げて「やり直し」を減らす

- Phase 0 で十分な学習データを投入
- やり直し率 < 5% を目指す

### 4. 写真の画質を適度に下げる

- 4K写真 → 1080p に圧縮する LINE 設定を活用
- ストレージ使用量が 1/4 に

## 🚨 予算超過を防ぐ仕組み

### GCP 予算アラート

月額 ¥10,000 を超えそうになったら、運営代表者のメールに自動通知。

```
設定場所: GCP Console → 課金 → 予算とアラート
閾値: 50%, 80%, 100%, 150% の4段階
```

### 自動停止 (オプション)

月額¥10,000を超えた時点で、Cloud Run を自動停止する設定も可能。
(ただし停止すると機能が使えなくなるので、通常は通知のみ推奨)

## 👥 運用体制

### 必要な人員

#### 最小構成 (1〜2名)

| 役割 | 担当 | 工数/月 |
|------|------|:-------:|
| 運営代表者 | 承認・月次確認 | **30分/月** |
| 補助担当 (Bus Factor対策) | 代行承認 | **5分/月** |

**合計: 月35分** で運用可能。

#### 推奨構成 (3〜5名)

| 役割 | 担当 | 工数/月 |
|------|------|:-------:|
| 運営代表 (Owner) | 全体統括・重要承認 | 20分 |
| 会計担当 (Staff) | レシート送信・確認 | 30分 |
| 受付担当 (Staff) | 名簿送信 | 10分 |
| 広報担当 (Staff) | 写真・SNS | 20分 |
| 技術サポート (Owner) | トラブル対応 | 0〜10分 |

## 📞 料金に関する問い合わせ

### GCP の請求書について

- 毎月1日〜末日の使用量を、翌月10日頃にメール送信
- Cloud Console → お支払い → 請求書でもDL可

### 不明な課金がある場合

- GCP お支払い → 明細 で内訳確認
- 不正請求の疑いがあれば Google サポートに連絡
- **ManabiOps 側の不具合でコスト超過した場合は、開発者と相談**
